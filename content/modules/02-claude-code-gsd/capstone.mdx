## The Project

Build and deploy a real, single-page web tool of your choosing using Claude Code and the GSD workflow. The tool must solve a real problem you have, be built entirely through Claude Code prompts (no manual coding), and end up live at a Vercel URL you can share. This is your first complete Claude Code build from blank repo to production.

## What You'll Build

- A working single-page web app on a topic you choose (examples: a personal finance tracker, a recipe idea generator, a daily habit checklist, a local event finder, a decision matrix tool)
- A GitHub repo with a clean commit history showing the build progression
- A live Vercel URL where the tool actually works
- A CLAUDE.md file in the project documenting the stack, conventions, and constraints
- A one-paragraph retrospective: what went well, what you'd do differently

## Skills You'll Use

- Lesson 2 (Install): Claude Code must be installed and authenticated
- Lesson 3 (First Project): The scaffold → run → verify loop drives the entire build
- Lesson 4 (GSD): Use /gsd:discuss-phase and /gsd:plan-phase before building
- Lesson 7 (CLAUDE.md): A CLAUDE.md must exist before the first build prompt
- Lesson 8 (Debugging): You will hit at least one error — debug it with Claude Code
- Lesson 9 (Git): Commit after each working milestone

## Getting Started

1. Decide on your tool: one sentence problem, one sentence solution. Run it through the Module 2 idea triage filter (Lesson 4 of Module 4 gives the full framework, but a quick gut check works here).
2. Create a new GitHub repo, clone it locally, write your CLAUDE.md, then run `claude` and start with /gsd:discuss-phase to scope the project.
3. Run /gsd:plan-phase to generate a build plan, then execute it with /gsd:execute-phase. Commit after each plan completes.

## Success Criteria

- [ ] A live Vercel URL exists for the tool and loads without errors
- [ ] The tool solves the problem you defined in Getting Started Step 1
- [ ] A GitHub repo exists with at least 5 commits showing build progression
- [ ] A CLAUDE.md exists in the repo with stack and conventions documented
- [ ] You debugged at least one error using Claude Code (using specific error description, not "it doesn't work")
- [ ] A one-paragraph retrospective is written (anywhere — notes app, text file, voice memo)

## Stretch Goals

- Add a second feature that requires a new GSD planning cycle
- Share the URL with one real person and incorporate one piece of feedback
- Write a second CLAUDE.md entry documenting a lesson you learned mid-build
