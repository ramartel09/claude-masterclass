## The Project

The GTA Hospital Wait Time Tracker is a real, deployed web app showing current emergency department wait times at Greater Toronto Area hospitals. When someone in the GTA needs urgent care, they can open this app, see which hospitals have shorter waits, and make an informed decision about where to go. This is the proof-of-mastery project — every skill from every module is used to build it. It's built the same way this masterclass was built: Claude Code + GSD workflow + zero manual coding.

## What You'll Build

- A live web app at a Vercel URL showing ER wait time data for GTA hospitals
- Data sourced from Ontario's public health aggregate data (verify source availability before Phase 4 begins — see Phase 4, Plan 04-01)
- A clean table or card layout showing: hospital name, location (borough/area), estimated wait time, and last updated timestamp
- Mobile-responsive design (readable on a phone while you're deciding where to go)
- A GitHub repo with a clean commit history built via the GSD workflow
- A link from this capstone page to the live deployed tracker (added in Phase 4, Plan 04-03)

## The Problem Being Solved

GTA emergency departments have dramatically different wait times — 1 hour at one hospital, 6 hours at another, at the same moment. This information exists but is buried in provincial health system pages that are hard to navigate on a phone in an emergency. The tracker surfaces it in a single, fast-loading, mobile-first view. Real value, real user, real problem. This is exactly the kind of tool the masterclass teaches you to build.

If the original data source isn't available, identify the best alternative (representative data, scraped public pages, or static demo data with a clear disclaimer).
2. **Plan 04-02**: Build the tracker using Claude Code + GSD workflow, starting from the verified data source.
3. **Plan 04-03**: Deploy to Vercel, add the live link to this capstone page, and write the final retrospective.

## Getting Started

1. Read Phase 4, Plan 04-01 — do not skip the data source verification step. Building a tracker without a data source is building a shell. The data is what makes it real.
2. Once data source is confirmed, write the REQUIREMENTS.md for the tracker using the format from Module 4, Lesson 3. Keep it tight: the MVP shows current wait times for 5–10 GTA hospitals and nothing else.
3. Start Claude Code, run /gsd:discuss-phase with the REQUIREMENTS.md, and follow the GSD workflow through to deployment.
